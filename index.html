Abstract
This study evaluates whether a redesigned e-commerce checkout page produces a measurable improvement in conversion rate (CR) over the existing design. Using a publicly available A/B testing dataset of 288,541 unique users distributed across three markets (US, UK, CA), we apply a five-method analytical pipeline: data preprocessing with mismatch filtering and deduplication, two-proportion z-test, non-parametric bootstrap resampling (10,000 iterations), country-level segmentation under Bonferroni correction, and logistic regression with country and time-bucket controls, including a heterogeneous treatment effect (HTE) interaction model. The new page shows a relative decrease of −1.30% in conversion rate (Z = −1.2949, p = 0.1953). The bootstrap 95% confidence interval [−3.24%, +0.71%] crosses zero, and all four logistic models yield a treatment odds ratio between 0.980 and 0.985 (p > 0.14). Country-level analysis reveals an asymmetric risk profile in Canada, where the 95% CI ranges from −14.29% to +3.11% — a 4.6× downside asymmetry. We conclude there is insufficient statistical evidence to roll out the new checkout page and recommend further iteration with explicit power analysis and a Bayesian re-evaluation framework for future experiments.

01 — Research Design

Hypothesis &
Experimental Setup

H₀ — Null Hypothesis
CR_new = CR_old
The redesigned checkout page produces no meaningful difference in conversion rate. Any observed difference is attributable to random variation.
H₁ — Alternative Hypothesis
CR_new ≠ CR_old
The redesigned checkout page produces a statistically significant difference in conversion rate — either positive or negative.
Parameter Value Description
Test Type Two-tailed Detects both improvement and degradation
Significance Level (α) 0.05 5% false positive rate threshold
Control Group old_page (group = control) Existing checkout design
Treatment Group new_page (group = treatment) Redesigned checkout page
Outcome Variable converted ∈ {0, 1} Binary: 1 = purchased, 0 = did not purchase
Pre-processing integrity: After mismatch filtering (removing users assigned to the wrong page variant) and deduplication, the treatment and control groups remained well-balanced: 144,226 control vs 144,315 treatment users. Country and time_bucket distributions were also nearly identical across groups, confirming randomization integrity post-cleaning.

06 — Final Verdict
Do Not Roll Out
the New Page
Across five independent analytical methods, the evidence consistently points to the same conclusion: the new checkout page does not demonstrate statistically reliable improvement.
01
Z-test p-value = 0.1953. The observed -1.30% relative drop is within the expected range of random variation. No statistical significance.
02
Bootstrap 95% CI = [-3.24%, +0.71%] crosses zero. Negative mean uplift is not statistically robust across 10,000 resampling iterations.
03
Canada market shows -5.92% uplift with asymmetric downside risk [-14.29%, +3.11%]. The potential loss is 4.6× the potential gain.
04
Logistic regression confirms treatment OR < 1 across all 4 models (basic, country-adjusted, time-adjusted, interaction), all non-significant.
Recommended Actions
→ Return the redesign to iteration
→ Investigate CA market specifically before any regional rollout
→ Define MDE and conduct power analysis before next test
→ Consider Bayesian A/B framework for future experiments
→ Run targeted tests per market with adequate sample sizes
07 — Limitations & Future Work

Study
Limitations

Missing Business Metrics
Revenue per user and Average Order Value (AOV) were not available. A page could convert fewer users but at higher value — this study captures only the binary conversion signal.
No Device / Traffic Source Data
Missing mobile vs. desktop splits and traffic source attribution. Simpson's Paradox could mask device-specific treatment effects that segment-level analysis would reveal.
Timestamp Limitations
The original timestamp format lacks calendar dates, preventing time-series analysis for novelty effects or day-of-week seasonality. Only elapsed session time was recoverable.
Future Work
Recommended next steps: Bayesian A/B testing framework for posterior probability analysis, sequential testing for early stopping, and multivariate testing on specific UI components.
